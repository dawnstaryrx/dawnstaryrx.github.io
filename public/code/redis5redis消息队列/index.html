消息队列

消息队列，Message Queue，字面意思就是存放消息的队列。最简单的消息队列模型包括三个角色：

  • 消息队列：存储和管理消息，也被称为消息代理Message Broker
  • 生产者：发送消息到消息队列
  • 消费者：从消息队列获取消息并处理消息

Redis提供了三种不同的方式来实现消息队列：

  • list结构：基于List结构模拟消息队列
  • PubSub：基于点对点消息模型
  • Stream：比较完善的消息队列模型
